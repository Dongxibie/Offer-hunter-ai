# 第一部分：项目报告

## 1. 一句话简介

**Offer Hunter AI** 是一个面向**学生 / 应届求职者**的 AI 求职助手 Web 应用：上传简历后，由大模型（DeepSeek）完成**简历分析、岗位匹配、简历优化、模拟面试、成长规划**五大核心能力。

## 2. 应用功能

| 模块 | 功能说明 | 后端接口 |
|------|----------|----------|
| 用户系统 | 注册 / 登录，JWT 鉴权（Token 有效期 24h），密码 BCrypt 加密 | `POST /api/user/register`、`POST /api/user/login`、`GET /api/user/info` |
| 简历上传与解析 | 上传 PDF / DOCX（≤10MB），用 PDFBox / POI 提取原始文本 | `POST /api/resume/upload`、`GET /api/resume/list`、`GET /api/resume/{id}` |
| AI 简历分析 | 提取技能标签、能力雷达评分、求职方向、综合评价 | `POST /api/ai/analyze-resume`、`GET /api/ai/analysis/{resumeId}` |
| 岗位匹配 | 简历与岗位库做匹配打分（技能分/项目分），列出匹配/缺失技能 | `GET /api/job/list`、`GET /api/job/{id}`、`GET /api/job/match?resumeId=` |
| AI 简历优化 | 针对目标岗位给出问题诊断 + 优化建议 + STAR 法则改写 | `POST /api/ai/optimize-resume?resumeId=&jobId=` |
| AI 模拟面试 | 按岗位生成技术/项目/HR/行为题，回答后 AI 点评打分 | `POST /api/interview/start`、`POST /api/interview/answer` |
| 成长规划 | 生成 30/60/90 天三阶段成长计划（目标/任务/学习资源） | `POST /api/ai/growth-plan?resumeId=` |

**前端页面**（Vue Router）：首页 `/`、登录 `/login`、上传 `/upload`、分析 `/analysis/:id`、岗位匹配 `/jobs`、简历优化 `/optimize/:resumeId/:jobId`、模拟面试 `/interview/:resumeId/:jobId`、成长规划 `/growth/:id`。除首页/登录/岗位列表外均需登录。

## 3. 技术栈

| 层 | 技术 |
|----|------|
| 前端 | Vue 3 + TypeScript + Vite 5 + Element Plus + Pinia + Vue Router + ECharts（能力雷达图） |

| 鉴权 | JWT（jjwt 0.12.5）+ BCrypt |
| 文件解析 | Apache POI 5.2.5（docx）、PDFBox 3.0.2（pdf）|
| 部署 | Docker + Docker Compose（mysql / backend / frontend 三容器），前端 Nginx 反代 `/api` |

# 第二部分：服务器部署攻略（详细教程）

目标：在一台 Linux 云服务器（推荐 Ubuntu 22.04，2 核 4G 起，AI 编译阶段建议 4G+ 内存或加 swap）上，用 Docker Compose 一键部署。

## 前置条件
- 一台公网 Linux 服务器（Ubuntu/Debian/CentOS 均可）。
- 已开放安全组端口：`80`（前端）、可选 `443`（HTTPS）。`3306/8080` 不建议对公网开放。
- 一个有效的 **DeepSeek API Key**（https://platform.deepseek.com 获取）。
- 域名（可选，做 HTTPS 时需要）。

## 步骤 1：安装 Docker 与 Compose
```bash
# Ubuntu/Debian
curl -fsSL https://get.docker.com | sh
sudo systemctl enable --now docker
docker compose version   # 确认 compose v2 可用
```

## 步骤 2：上传项目到服务器
方式 A（推荐，本地打包上传）：
```powershell
# 在本地 Windows（排除 node_modules / target 以减小体积）
cd C:\Users\PC\Projects
tar --exclude=offer-hunter-ai/frontend/node_modules --exclude=offer-hunter-ai/backend/target `
    -czf offer-hunter-ai.tar.gz offer-hunter-ai
scp offer-hunter-ai.tar.gz user@<服务器IP>:/opt/
```
服务器解压：
```bash
cd /opt && tar -xzf offer-hunter-ai.tar.gz && cd offer-hunter-ai
```
方式 B：推送到 Git 远程后在服务器 `git clone`。

## 步骤 3：配置环境变量（关键）
编辑项目根目录 `.env`：
```bash
nano /opt/offer-hunter-ai/.env
```
填入真实值（务必修改三项）：
```env
DEEPSEEK_API_KEY=sk-你的真实key
MYSQL_PASSWORD=改成强密码
JWT_SECRET=改成一段长随机字符串
```

## 步骤 4：一键构建并启动
```bash
cd /opt/offer-hunter-ai
docker compose up -d --build
```
- 首次构建会下载 Maven/npm 依赖，耗时较长（5~15 分钟）。
- MySQL 容器首次启动会自动执行 `database/schema.sql` 与 `init-data.sql` 建表灌数据。

## 步骤 5：验证
```bash
docker compose ps                 # 三个容器应为 healthy/running
curl http://localhost/api/job/list   # 应返回 12 条岗位 JSON
docker compose logs -f backend    # 看后端启动日志
```
浏览器访问 `http://<服务器IP>`，用 `testuser / 123456` 登录测试全流程。

## 步骤 6（可选）：配置 HTTPS + 域名
推荐在三容器外再加一层反向代理（或改 `frontend/nginx.conf`）。最简方案用 Caddy：
```bash
# /opt/Caddyfile
your-domain.com {
    reverse_proxy localhost:80
}
```
```bash
docker run -d --name caddy --network host \
  -v /opt/Caddyfile:/etc/caddy/Caddyfile \
  -v caddy_data:/data caddy
```
Caddy 会自动申请并续期 Let's Encrypt 证书。

## 常用运维命令
```bash
docker compose restart backend        # 重启后端
docker compose down                   # 停止（保留数据卷）
docker compose down -v                # 停止并删除数据（慎用，清库）
docker compose up -d --build backend  # 改代码后重建单个服务
docker compose logs -f                # 看全部日志
docker volume ls                      # 数据卷：mysql_data / uploads_data
```

## 排错速查
| 现象 | 可能原因 / 解决 |
|------|----------------|
| 后端起不来，连不上 DB | MySQL 未 healthy；查 `MYSQL_PASSWORD` 是否与 compose 一致 |
| AI 功能报错 / 超时 | `DEEPSEEK_API_KEY` 无效或余额不足；服务器无法访问 api.deepseek.com |
| Maven 构建失败 | 拉不到 `repo.spring.io/milestone`（里程碑版）；检查服务器网络/代理 |
| 前端 404 / 接口跨域 | Nginx `/api` 反代未生效；确认 frontend 依赖 backend healthy |
| 内存不足构建被 kill | 加 swap：`fallocate -l 4G /swapfile && mkswap /swapfile && swapon /swapfile` |
| 端口 80 被占用 | 改 `docker-compose.yml` frontend 端口映射，如 `"8081:80"` |
